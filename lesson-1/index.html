Git команди:

git config --global user.name - команда, що встановлює ім`я користувача в системі Git;

git config --global user.email - команда, що встановлює адресу електронної пошти користувача в системі Git;

git init - команда, що створює новий підкаталог .git, який містить всі необхідні файли репозиторія;

git status - команда, що показує стан файлів( які файли змінились, контрольовані та неконтрольовані Gitом файли, в якій гілці знаходимось і т.д.)

git add - команада, яка вказує Git що файл потрібно відслідковувати;

git commit - команда для створення коммітів;

git log - команда, що покаже зроблені коміти репозиторія;

git checkout - команда, щоб перейти на іншу гілку;

git branch - команда, що створює нову гілку;

git push -команда, що відправляє комміти з локального на віддалений репозиторій;

git pull - команда, що скачує всі зміни з віддаленного репозиторію і встановлює їх відразу в локальному;

git merge - використовується для об`єднання(злиття) даних кількох гілок в одній поточній;

git clone - команда для клонування Git-репозиторія;

git revert - команда, що скасовує зміни комміта, шляхом створення нового без цих змін;

git reset - команда для відміни локальних змін в репозиторії Git;

git rebase - команда за допомогою якої, можна взяти всеі комміти з одної гілки і втому ж порядку застосувати їх до другої;

git cherry-pick - команда, що бере певні комміти з одної гілки і копіює їх в поточну;

git stash - переміщає поточні зміни до спеціального сховища, тобто приховує ці зміни, зберігаючи їх окремо, щоб повернути пізніше, коли це знадобиться;

.gitignore файл - файл, в якому містяться файли-виключення, що не мають пушитись і передаватись на віддалений репозиторій;
